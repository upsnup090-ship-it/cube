# Review Workflow

Use this workflow to review existing code or a proposed change.

## Step 1: Determine scope
Identify what should be reviewed and what is out of scope.

## Step 2: Inspect
Check:
- correctness
- regressions
- security risks
- error handling
- unnecessary complexity
- missing tests

## Step 3: Rank findings
Group findings:
- Critical
- Important
- Minor
- Suggestions

## Step 4: Report
For each finding include:
- file/location
- problem
- why it matters
- suggested fix
