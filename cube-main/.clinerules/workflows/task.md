# Task Workflow

Use this workflow for a normal implementation task.

## Step 1: Understand the task
Restate the goal in 2-5 lines. Identify the expected final result.

## Step 2: Inspect context
Read the project start files if present:
- `00_START_HERE.md`
- `AGENTS.md`
- `README.md`
- relevant files from `06_FILE_MAP.md`

## Step 3: Plan
List:
- files likely to be changed
- risks
- smallest safe implementation path

## Step 4: Implement
Make only the changes required for this task.

## Step 5: Verify
Run the smallest relevant check:
- tests
- lint
- build
- manual command from `08_COMMANDS.md`

If verification cannot be run, explain why.

## Step 6: Report
Summarize:
- files changed
- what changed
- why
- how to check
- remaining risks
