# Code Style

## General
- Code must be readable and boring.
- Prefer explicit names over clever shortcuts.
- Keep functions focused.
- Avoid duplicated business logic.
- Avoid unnecessary abstractions.

## Errors
- Handle expected errors explicitly.
- Do not swallow exceptions silently.
- Error messages should help locate the problem.

## Configuration
- Do not hardcode secrets.
- Put environment-specific values into config.
- Keep config changes minimal and documented.
