# Setup Project Workflow

Use this workflow when opening a new or unknown project.

## Step 1: Scan root
Identify stack, package manager, entry points, and important docs.

## Step 2: Read docs
Read:
- `README.md`
- `AGENTS.md`
- `00_START_HERE.md`
- package/config files

## Step 3: Build file map
Update `06_FILE_MAP.md` with real project folders and important files.

## Step 4: Build commands list
Update `08_COMMANDS.md` with real install/run/test/lint/build commands.

## Step 5: Identify risks
Update `context/known_issues.md` and `context/constraints.md` if useful.

## Step 6: Report
Summarize how the project is structured and how to run/check it.
