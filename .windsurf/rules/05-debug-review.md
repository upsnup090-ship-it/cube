# Debug and Review Rules

## Debug sequence
1. Reproduce or clearly describe the problem.
2. Locate the smallest failing area.
3. Explain the suspected cause.
4. Apply a minimal fix.
5. Verify the fix.

## Review sequence
- Separate critical issues from minor improvements.
- Check for regressions and side effects.
- Check error handling.
- Check whether the solution is simpler than the problem.
