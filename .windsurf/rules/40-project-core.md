# Project Core

## Project Identity

- Project: BigPlayBot Dices / CubeChat
- Product type: Telegram-first PvP dice game foundation
- Current MVP focus: PvP Dices only with safe internal balance accounting

## MVP Scope (Current)

- Telegram user identity model
- Internal wallets with available and locked balances
- Escrow lock flow for PvP games
- Dice roll recording with auditability
- Settlement and refund flows
- Immutable ledger entries
- Admin monitoring and audit logs

## Out of Scope (Unless Explicitly Requested)

- Banker mode
- Real payment processors
- Crypto deposits or withdrawals
- Fiat deposits or withdrawals
- Referrals, bonuses, promotions
- Tournaments
- Production gambling/compliance features

## Working Principle

- Make minimal, safe, task-focused changes.
- Prefer small, reviewable edits over broad rewrites.
- Do not refactor unrelated areas during scoped tasks.
