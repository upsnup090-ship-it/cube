# Core Project Rules

## Language
- Work in Russian unless the user asks otherwise.
- Keep explanations practical and short.
- Do not hide assumptions; list them explicitly when the task is unclear.

## Default behavior
- First understand the task and inspect the relevant project files.
- Do not edit files until the goal, affected files, and risks are clear.
- Prefer minimal targeted changes over broad rewrites.
- Do not do unrelated cleanup "while you are here".
- Before risky or large changes, present a short plan.

## Required response format after changes
1. What changed
2. Files changed
3. Why it was changed
4. How to check
5. Risks / notes
